恭喜你！如果你已經讓 **Pixel 10** 和 **Raspberry Pi** 都在 Tailscale 的清單中看到彼此，你已經完成了最難的「互連」步驟。

要讓 Pixel 10 透過樹莓派上網（也就是把樹莓派當作你的專屬出口），這在技術上稱為 **「出口節點 (Exit Node)」**。這非常實用，因為不管你在哪裡，你的網路流量都會先回到家裡的樹莓派，再從家裡連出去，就像你人坐在家裡客廳上網一樣。

請按照以下三個簡單的步驟進行設定：

---

### 第一步：在樹莓派上開啟「出口節點」功能

因為樹莓派預設只會收發自己的封包，你必須告訴它：「請幫我的其他裝置轉發流量」。

1. **修改系統設定（允許轉發）：**
輸入這兩行指令，確保 Linux 系統核心允許封包轉發：
```bash
echo 'net.ipv4.ip_forward = 1' | sudo tee -a /etc/sysctl.conf
echo 'net.ipv6.conf.all.forwarding = 1' | sudo tee -a /etc/sysctl.conf
sudo sysctl -p

```


2. **啟動 Tailscale 出口服務：**
輸入以下指令重新啟動 Tailscale：
```bash
sudo tailscale up --advertise-exit-node

```



---

### 第二步：在 Tailscale 管理後台「核准」它

為了安全，Tailscale 不會讓你一設定完就自動生效，你必須去網頁後台點個頭。

1. 用電腦或手機登入 [Tailscale Admin Console](https://login.tailscale.com/admin/machines)。
2. 找到你的 **Raspberry Pi**，點擊右邊的「三個點」或「Edit route settings」。
3. 勾選 **Exit Node** 選項並儲存。

---

### 第三步：在 Pixel 10 上選擇使用該節點

現在，你的樹莓派已經像是一個廣播電台準備好發號施令了，我們讓手機接收它。

1. 打開 Pixel 10 上的 **Tailscale App**。
2. 在主選單找到 **"Exit Node"** 的選項（通常在頂部或點擊右上角選單）。
3. 從清單中選擇你的 **Raspberry Pi**。
4. 開啟連線開關。

---

### 🧐 怎麼知道成功了沒？

最簡單的測試方法：

1. 在 Pixel 10 上打開瀏覽器，搜尋「**My IP**」。
2. 看看顯示的 IP 地址。如果它變成了你**家裡的寬頻 IP**（而不是手機 5G 的 IP），那就代表你成功透過樹莓派「翻」回家裡上網了！

### 💡 老師的進階小筆記 (后設認知策略)

當你開啟 Exit Node 時，你會發現手機的網速可能會變慢一點。這不是因為 Tailscale 不給力，而是受限於兩個物理因素：

* **家裡的「上傳速度」**：樹莓派要把抓到的網頁傳給你，用的是家裡寬頻的「上傳頻寬」。
* **來回延遲**：數據從 5G 基站 -> 家裡樹莓派 -> 目標網站 -> 回家裡 -> 回手機，這條路徑比直接上網長得多。

**如果你成功連上了，試試看在 Pixel 10 上打開一個只有在家裡才能看的服務（例如家裡的 NAS 網頁或是印表機管理介面），看看是不是變得很順暢？**

連線過程中如果有跳出什麼錯誤訊息（例如手機顯示連線中斷），隨時告訴我，我們來看看是不是防火牆規則需要微調！你想現在就去試試看嗎？
