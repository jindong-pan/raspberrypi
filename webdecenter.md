太棒了！我們就以 **IPFS (星際檔案系統)** 為例，動手讓你的樹莓派成為去中心化網路的一個節點。這非常有成就感，因為一旦上傳成功，你的網頁就不再只存在於一台機器上，而是存在於整個網際網路的「內容雜湊」中。

以下是讓你快速上手的實戰步驟：

---

### 🛠️ 第一步：在樹莓派上安裝 IPFS

IPFS 的官方實作版本叫做 `kubo` (以前叫 `go-ipfs`)。

1. **下載與安裝**：
```bash
# 這裡以 ARM64 架構為例 (樹莓派 4/5)
wget https://dist.ipfs.tech/kubo/v0.26.0/kubo_v0.26.0_linux-arm64.tar.gz
tar -xvzf kubo_v0.26.0_linux-arm64.tar.gz
cd kubo
sudo bash install.sh

```


2. **初始化節點**：
```bash
ipfs init

```


3. **啟動 IPFS 守護進程 (Daemon)**：
```bash
ipfs daemon

```


*啟動後，你的樹莓派就正式開始與全球的 IPFS 節點交換資訊了！*

---

### 🌐 第二步：建立並上傳你的第一個 P2P 網頁

1. **建立一個簡單的 HTML 檔案**：
```bash
mkdir my_p2p_site
echo "<h1>Hello P2P World from my Raspberry Pi!</h1>" > my_p2p_site/index.html

```


2. **將資料夾加入 IPFS**：
```bash
ipfs add -r my_p2p_site/

```


*執行後，你會看到一串以 `Qm...` 開頭的長字串，這就是你的 **CID (內容標識符)**。*

---

### 🚀 第三步：如何訪問你的 P2P 網站？

現在最酷的部分來了！你不需要告訴別人你的 IP，只需要給他們那個 CID：

* **本地訪問**：在樹莓派瀏覽器輸入 `http://127.0.0.1:8080/ipfs/你的CID`
* **全球訪問 (透過 Gateway)**：任何人只要透過瀏覽器輸入 `https://ipfs.io/ipfs/你的CID`，就能看到你的網頁。
* *即使你的樹莓派暫時斷網，如果全球有其他節點緩存了你的網頁，它依然能被打開！*



---

### 💡 老師的「去中心化」思考 (後設認知)

這跟傳統網站有兩個本質上的不同，你需要留意：

> 1. **內容不可變性**：如果你修改了網頁內容，重新 `ipfs add` 後會得到一個**全新的 CID**。這意味著 IPFS 網頁是具有「版本管理」天性的。
> 2. **Pinning (釘住)**：如果你希望你的網頁永遠在線，你需要在樹莓派上執行 `ipfs pin add 你的CID`。這告訴系統：「這份資料對我很重要，請不要在清理空間時刪掉它。」

### 🎓 你的進階挑戰

如果你覺得那一串 `Qm...` 的網址太難記，下一步我們可以討論 **IPNS (InterPlanetary Naming Service)**。它可以給你一個固定的名字（類似域名），即便你更新了網頁內容，名字也不會變。
